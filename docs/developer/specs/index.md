# 开发者规范索引

这里收录维护者使用的规范文档。它们会被构建进站点，但默认不出现在公开导航栏中。

## 使用顺序

1. 先确定 `change slug`
2. 新建或更新 `docs/developer/specs/<slug>.md`
3. 再新建或更新 `docs/developer/design/<slug>.md`
4. 最后实现代码、测试与公开文档
